# P1 数字教材完整样例需求

## 1. 产品目标

DGBook 当前只证明一条完整主线：

```text
角色登录 -> 当前学习/授课入口 -> P1项目 -> P01/P02/P03任务
-> 能力节点 -> 教材学习或课堂跟随 -> 微练习 -> 正式测试
-> 专业成果 -> 教师复核 -> 项目成果包 -> 能力图谱达成
```

课程能力图谱负责解释课程、岗位、任务和能力之间的关系，但不再作为学生或教师登录后的默认首页。

## 2. 本轮范围

- 桌面优先，验收视口为 `1440x900` 与 `1920x1080`。
- 完整开放 P1 的 P01、P02、P03；P2-P6 只展示整书结构，不开放学习。
- P1T1-N02 是本轮深度教材标杆，必须支持脱离教师和语音后的完整自学。
- 教师、学生、投屏使用同一课堂会话和学习数据事实源。
- 保留固定讲师张老师、OpenMAIC 式单向播报和专业 PixiJS 正式测试。

## 3. 非目标

- 不建设生成、治理、审核后台或AI生产链页面。
- 不增加聊天、问答导师、圆桌讨论或多讲师模式。
- 不改造旧 Astro 教材，不手改 `site/dist/`。
- 不把尚未完成的 P2-P6 包装成可学习内容。

## 4. 角色默认入口

### 4.1 学生首页

学生登录后进入学习首页，30秒内回答：

1. 我正在学什么。
2. 为什么要学。
3. 下一步做什么。
4. 做到什么算完成。

首页必须显示当前项目、当前任务、当前能力节点、完成标准和唯一主按钮“继续学习”。“查看其他任务”和“课程能力图谱”为次级入口，教师与投屏入口不得出现。

### 4.2 教师工作台

教师登录后进入授课工作台，显示当前课程、演示班级、最近授课位置、继续授课、开始新课、待批阅专业成果、班级薄弱点和课程能力图谱入口。

教师应在两次点击内进入 P1T1-N02 授课页。工作台不要求教师先理解完整图谱。

## 5. P1完整项目

| 任务 | 任务主题 | 任务级专业成果 |
| --- | --- | --- |
| P01 | 室内信息采集 | 《室内设备与链路证据表》 |
| P02 | 室外信息采集 | 《室外站点与覆盖采集表》 |
| P03 | 投诉信息采集 | 《投诉信息调查单》 |

三个任务各包含四个可学习能力节点、完整自学内容、微练习和至少一个5分钟以上的PixiJS正式测试。任务终点节点提交专业成果并进入教师复核。

三份认证成果自动汇编为《5G网络信息采集成果包》，包含封面、版本、证据索引、风险清单、教师评语、任务综合分和项目综合分，并支持查看与导出。

## 6. P1T1-N02深度教材

保留“问题、看图、步骤、纠偏、练习、成果”六段结构，但正文必须形成可独立学习的连续教材章节，包含：

- 案例背景、任务问题和必要前置知识。
- 术语解释及可查询词条。
- 带标注的设备图和证据定位图。
- 设备位置、设备身份、连接方向三类判断依据。
- 可复述的逐步推理过程。
- 至少两个完整正例和两个完整反例。
- 基础、应用、迁移三级练习。
- 针对错误原因的反馈和改正路径。
- 迁移任务、专业成果模板和明确评价量规。

关闭教师讲解和语音后，非专业学习者仍应能够解释三类证据分别是什么，以及为什么需要它们。

## 7. 自学与课堂跟随

### 自学模式

- 使用完整教材正文。
- 学生可自由前后阅读、查术语、做案例、练习、重试和提交成果。
- 个人阅读位置和完成状态由学生学习记录维护。

### 课堂跟随模式

- 只展示教师当前讲授页、课堂位置、教师任务和当前活动。
- 跟随学生接收教师场景、页码、播放与活动切换。
- 自主学习中的学生只收到“教师已更新课堂位置”提示，不被强制跳页。
- 提供返回完整自学内容和“一键回到教师当前页”。

两种模式共享同一教材内容定义，但使用不同渲染器和不同游标，禁止互相覆盖进度。

## 8. 统一学习状态机

权威状态顺序为：

```text
未解锁 -> 可学习 -> 学习中 -> 微练习通过 -> 正式测试达标
-> 专业成果已提交 -> 待教师复核
-> 退回修订 -> 待教师复核
-> 教师认证 -> 能力达成
```

节点配置必须显式声明：

- 是否要求正式测试。
- 是否要求专业成果。
- 是否要求教师认证。
- 解锁所需的前置节点。

普通节点可依据配置跳过成果和教师认证；P01-N04、P02-N04、P03-N04必须完成任务级成果和教师认证。任何前端不得自行推断或缩短状态链。

“测试达标”“待认证点亮”“教师认证”“能力达成”必须使用不同文案和视觉状态。

## 9. 成绩口径

- 节点测试最高分：节点正式测试重试后的最高成绩。
- 任务Pixi正式测试分：任务旗舰关卡成绩。
- 任务综合分：节点测试平均分20% + Pixi正式测试30% + 专业成果50%。
- 项目综合分：P01、P02、P03任务综合分等权平均。

教师认证前只显示任务暂估分。正式测试100分不能替代专业成果或教师认证。

## 10. 路由和权限

- 路由解析不得把未知节点静默回退到 P1T1-N01。
- 不存在的节点显示“不存在”；尚未开放的节点显示“尚未开放”。
- 被锁定节点只显示前置要求，不加载教材正文、练习、测试或提交组件。
- UI锁定状态、路由门禁和权威学习状态必须来自同一规则计算。

## 11. 三端单一事实源

现有课堂会话API和Helper构成跨设备事实源：

- 服务端保存课堂场景、命令修订、设备心跳、命令回执和学习快照。
- 教师端只在收到真实 `applied` 回执后显示控制成功。
- Helper离线时明确显示“课堂同步不可用”，不得用预置名单伪造在线或已跟随。
- 学生、教师、投屏和图谱均读取同一权威投影，不分别计算提交数或成绩。
- 投屏只读取匿名聚合，不读取教师私有脚本和学生个人答案。

## 12. 验收基线

- 学生登录后30秒内找到正确学习入口。
- 教师两次点击内进入P1T1-N02授课页。
- P01、P02、P03可按顺序完成并形成项目成果包。
- 非专业人员可独立学懂P1T1-N02的三类证据逻辑。
- 跟随学生同步，自主学习学生不被覆盖进度。
- 同一状态在学生、教师和图谱端文案一致。
- 锁定、未知和未开放节点均无法绕过。
- 同时刷新三端时，在线人数、提交数、分数和状态一致。
- 所有正式Pixi测试可失败、复盘、重试和达标，且成绩进入教师视图。
