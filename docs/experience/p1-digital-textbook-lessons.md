# P1数字教材迭代经验

## 产品经验

1. 数字教材首先要让学习者学懂并完成职业任务，不能用平台功能数量替代教材深度。
2. 能力图谱是课程结构和学习证据的解释层，不是学生与教师的默认操作首页。
3. 角色首页必须围绕下一步行动设计：学生继续学习，教师继续授课或处理待复核成果。
4. 资源、动画、练习和游戏只有挂接到能力节点、学习动作和评价成果时才有意义。
5. 自学与课堂跟随应共享内容定义，但不能共享同一个阅读游标或完全相同的页面密度。
6. “完成练习”“测试达标”“教师认证”“能力达成”是不同事件，禁止都叫“点亮”。

## 教材经验

1. 六段式结构只是导航骨架，不等于教材内容；每段需要案例、依据、推理、正反例和反馈。
2. 专业图形应承担证据解释任务，连线必须连接真实端点，标签不得遮挡关系。
3. OpenMAIC式播报应把字幕、焦点、指示动作和教材段落绑定，播报栏不能覆盖正文。
4. 微练习用于即时反馈，正式Pixi测试用于成绩；两者的次数、分数和状态必须分开。
5. 职业教育的最终证据是可复核的专业成果，不是选择题满分或游戏通关动画。

## 课堂经验

1. 教师控制成功必须以设备心跳和命令回执为依据，不能由前端乐观更新或预置名单推断。
2. 教师、学生、投屏应共享同一个场景修订号；角色差异只体现在可见字段和可执行动作。
3. 教师私有脚本、学生个人答案和身份信息不得出现在投屏端。
4. 自主学习者可感知课堂更新，但不能被教师命令覆盖个人阅读位置。

## 数据经验

1. 课程配置、学习事件、状态投影和课堂快照需要明确分层。
2. 页面只消费权威读模型，不在组件中重复推导解锁、提交数、通过率或综合分。
3. 节点最高分、任务综合分和项目综合分必须使用不同字段和标签。
4. 路由门禁必须调用与界面锁定相同的策略函数，未知ID不能静默回退。

## 工程与资源经验

1. 直接在 `D:\Claude\dgbook` 工作，不为审查或开发创建完整仓库副本。
2. TEMP、npm缓存和Playwright浏览器统一落在D盘；不得在C盘临时目录复制Git对象、依赖或构建输出。
3. `output/`、`artifacts/` 和运行日志均为可再生文件，完成里程碑后清理。
4. 日期型计划、Agent简报和过程日志不长期保留；当前需求更新到唯一需求文档。
5. 清理Git对象前必须验证绝对路径、工作树状态、分支与reflog可达性，并先运行连接性检查。
