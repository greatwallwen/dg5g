# DGBook Ralph Loop 迭代契约

## 极简 Goal

> 每轮完成并上线一个最短可验证纵向切片，使 1 名教师、3 名学生和能力图谱共享 SQLite 真实状态并对齐 Image2；本地门禁与远端冒烟失败就修，成功后选择下一项最高优先未满足验收，直到八项需求全部有线上证据。

这句话是当前唯一执行命令。详细设计以 P1 数字教材样张、Image2 视觉标准、SQLite 真实状态和一师三生演示闭环为约束，不把 Ralph Loop 当作缩小范围或绕过验收的理由。

## 深度反思

前几轮证明，大量横向并行可以快速生成文件，却也容易制造四类假进度：

1. 模块测试通过，但演示账号仍是不可登录的占位散列；
2. 新策略存在，但旧组件仍按节点编号、数组位置或本地阈值重造规则；
3. P03 数据已经生成，路由却仍静默回落 P01；
4. 多代理共享树中的 RED 被误读成别人的回归，整体门禁迟迟没有稳定检查点。

因此后续不再用“文件完成”衡量进度，而只承认可从登录一路走到结果、刷新后仍一致、发布后仍可复现的纵向证据。所有派生界面必须消费同一服务端投影；模拟数据只允许来自同一套 SQLite demo seed；不存在、锁定、未认证都必须失败关闭。补考、审批和大班规模不进入当前关键路径。

每次 Review 同时回答三个反思问题：产品是否让学生或教师更快完成真实任务；创新是否由 Image2、证据推理、课堂同步或交互测试形成可验证增益；完整性是否覆盖数据、授权、路由、UI、发布和远端行为，而不是只覆盖组件。

## Review

- 从八项用户验收中选择尚无强证据且最阻塞学生、教师或图谱主旅程的一项。
- 先检查当前代码、SQLite、运行页面和远端版本，不凭报告或记忆判断完成。
- 检查上一轮是否遗留占位值、第二套 ID、静默 fallback、客户端授权、写死人数/分数或临时文件。
- 若同一失败跨端出现，优先修事实源或投影，不分别修三个页面。

## Analyze

- 写出本轮唯一用户结果、权威输入、状态触发条件和可观察输出。
- 把问题分成数据事实、授权边界、路由门禁、展示模型和视觉层，找最上游根因。
- 自主确定实现细节；只有业务语义会改变、需要新外部权限或存在高风险不可逆操作时才请求确认。
- 一轮最多保留一个主切片和必要的阻断修复，防止重新变成横向大包。

## Layout

- 先写最小失败测试，覆盖真实角色、3 名学生、canonical ID、状态和路由。
- 明确本轮文件所有权；并行代理不得同时编辑同一高冲突文件。
- 预先定义完成证据：focused tests、全量 unit/typecheck/build、两种 viewport、远端 URL、SQLite 计数与跨端 snapshot version。
- Image2 只允许深海军蓝工程壳、青色当前、绿色达成、黄色复核/风险、红色真实故障；主任务居中，图谱为次级入口，圆角不超过 8px。

## Produce

- 严格 RED → GREEN；先修根因，再接页面。
- 使用 importer 作为教材生成源，使用 SQLite/service 作为运行事实源，使用 canonical policy/projector 作为学习状态源。
- 禁止 localStorage 身份、客户端教师认证、URL 绕过、未开放节点回落、分别写死三端数字。
- 每个阶段本地门禁通过后，使用会话内 SSH 凭据原子发布到 `8.153.206.97`；口令不得写入仓库、日志或报告。

## Harden

- 运行当前切片 focused tests，再运行 `pnpm web:test:unit`、`pnpm web:typecheck`、`pnpm web:build` 和适用审计；涉及 EduGame 时必须包含 `audit:edugame-runtime`。
- 运行 `pnpm validate:ralph-loop` 与 `qa:gates` 中适用且真实存在的门禁；缺失脚本必须被记录和修复，不能当作通过。
- 发布前备份并验证持久 SQLite；发布失败恢复旧 current；发布后分别以教师、3 名学生、投屏和图谱做远端冒烟。
- 比较同一 snapshot 的人数、提交数、节点测试最高分、任务综合分、项目综合分和课堂位置。
- 保存截图与机器可读报告；清理 `__pycache__`、`.pyc`、临时数据库、截图草稿和旧 release 包。

## 循环与终止

Harden 全绿才结束本轮。随后回到 Review，自主挑选下一项最高优先缺口。
任一门禁失败则仍在同一轮，不把目标改写成较小的“兼容版本”。只有下列条件同时成立才可结束整个 Ralph Loop：八项用户验收均有当前本地证据和线上证据；学生、教师、投屏、图谱同源；P01/P02/P03 与 P1 成果包可走通；P1T1-N02 脱离教师/语音仍能独立学懂；Image2 两种 viewport 通过；远端刷新与重启后 SQLite 状态保持一致。
