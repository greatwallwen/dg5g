# DGBook 产品收口 Ralph Loop

## 当前阶段反思

### 产品层面

当前阶段坚持 5G 优先收尾。学生默认入口、教师授课工作台、P1 三任务、能力图谱和课堂跟随必须形成同一条真实旅程；补考、复杂审批、24 人大班和通用多课程平台不在下一轮展开。

### 创新维度

创新不以功能数量衡量，而以 Image2 工程视觉、可解释设备证据链、完整自学内容、课堂同步但不覆盖个人进度、以及 Pixi 正式测试与专业成果的清晰分工衡量。所有创新都必须落在真实 SQLite 状态和可复核职业成果上。

### 完整性维度

完整性要求 Review、Analyze、Layout、Produce、Harden 五阶段闭环。每轮运行 focused tests、全量 Web gates、`qa:gates` 中适用检查、`audit:edugame-runtime`、浏览器旅程和远端冒烟；只有页面、数据、权限、路由和统计同时一致才算完成。

### 可迁移性维度

教材内容继续由 importer/schema 生成，运行状态由 SQLite repository/service 提供，UI 消费稳定 DTO。P1 的课程语义允许专用配置，但认证、状态、门禁、快照和发布机制不得硬编码为 P1 私有实现。

### 下一轮重点

完成真实登录与双角色默认首页的线上纵向切片；随后按门禁、P1 内容、课堂四端同步和 Image2 视觉顺序循环收口。每一轮都先解决最高优先未验证用户结果，不以新增文件数量作为进度。
