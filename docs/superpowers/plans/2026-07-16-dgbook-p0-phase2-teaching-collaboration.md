# DGBook P0 Phase 2：授课、协同与图谱真实挂接计划

## 目标与边界

在 Phase 1 已建立的真实学习、正式测试、专业成果与项目成果包事实之上，完成 P01 两课时授课样张、教师/投屏/课堂跟随协同和能力图谱真实挂接。维持一个教师、三个学生、P1 三任务十二节点；不扩展补考、审批或其他项目。

## Task 9：建立 P01 两课时权威授课包

- 用单一 TypeScript 内容模型定义两课时、建议每课时 45 分钟、合计约 12 个授课页。
- 每页必须包含：建议时间、投屏内容、教师讲解、案例化提问、具体典型答案、常见错误、追问方式、学生动作、过渡语。
- 典型答案必须引用 P01 室内设备与链路案例，禁止再次使用通用占位句。
- 单元测试检查两课时总时长、全部字段、页序与 P1T1-N02 关键证据教学覆盖。

## Task 10：接入教师授课工作台

- 教师工作台继续保持课程、班级、最近授课位置和继续授课入口。
- 授课页显示课时、当前页、建议时间、教师私有讲稿和案例答案；投屏只显示共同内容。
- “继续授课”在两次点击内到达 P1T1-N02 对应授课页；上一页/下一页使用服务端课堂位置。
- 不把教师脚本、学生身份或个人答案泄露到投屏端。

## Task 11：收紧课堂状态与讲评门禁

- 课堂助手离线时，同步类操作必须禁用，并提供明确重连动作或错误说明。
- “进入讲评”必须由服务端检查当前正式测试/课堂活动提交数；0 份提交返回明确错误，前端同步禁用。
- 教师切换页推进权威课堂 revision；投屏与 follow 学生读取同一 revision。
- self 学生只收到“教师有新位置”提示，不覆盖个人阅读游标。

## Task 12：补齐投屏控制与匿名讲评

- 投屏端提供全屏、上一页、下一页、返回教师端；翻页写入与教师端相同的服务端会话命令。
- 上一页/下一页遵守发布边界与 revision 冲突检测，不在浏览器本地伪造页码。
- 讲评只显示匿名错误维度、人数与比例，不返回学生姓名、账号、答案或证据详情。
- 浏览器测试覆盖教师、两个 follow/self 学生和投屏的跨上下文同步。

## Task 13：修复能力图谱真实挂接

- 正式测试节点只进入独立 `/learn/[nodeId]/test` 页面。
- 专业产出节点只进入当前学生、当前任务的真实成果页。
- 节点、任务与项目分数分别来自严格校验后的测试、当前版本教师评价和三任务认证。
- `能力达成` 只由真实学习、练习、正式测试、成果提交和教师确认事件产生；演示事实必须显示演示标识。

## Task 14：整体验收、发布与同步

- 运行 637+ 单元测试、typecheck、结构检查、生产构建、Phase 1 专项审计、媒体 56 项门禁与完整 `qa:gates`。
- 运行三学生、教师、投屏、课堂 follow/self、正式测试、成果退回/修订/确认、成果包和图谱浏览器矩阵。
- 生成不含数据库与凭据的源码发布包，原子部署到 `8.153.206.97`，核验 build-info、登录和受保护页面。
- 远程冒烟通过后，将最终分支推送到 `git@github.com:greatwallwen/dg5g.git`。

## 完成判定

只有本地门禁、浏览器矩阵、远程冒烟与 GitHub 推送全部通过，Phase 2 才可完成；任何预置数据、客户端分数或仅靠页面打开产生的状态都不能替代真实事件。
