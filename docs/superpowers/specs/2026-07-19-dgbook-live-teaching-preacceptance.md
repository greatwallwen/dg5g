# DGBook P1 线上试教预验收规格

**状态：** 已批准

**批准日期：** 2026-07-19

**适用范围：** 当前 DGBook P1 四课时教学样张的下一轮线上收敛
**前置版本：** `docs/superpowers/specs/2026-07-17-dgbook-p1-classroom-ready-sample-design.md`

## 1. 目标

冻结新增功能，以一次可重复的真实教学操作流程验证当前样张。当前轮次证明“教师和三名学生能自然完成课堂操作”，不把知识迁移效果设为通过条件。

当前预验收使用一台电脑上的独立浏览器上下文，以提高演示、定位和重跑速度。它属于演示级预验收；正式对外宣称真实课堂可用前，仍需补做教师电脑、独立投屏和三台学生设备的多设备验证。

## 2. 人员与准备

- 一名教师，使用 `teacher01`。
- 三名非专业学生，分别使用 `student01`、`student02`、`student03`。
- 教师课前允许十分钟阅读现有使用教程、登录并查看授课工作台。
- 正式四十五分钟课堂开始后，研发人员不得提示、代操作、输入内部 URL、修改数据库或调用业务 API 修复状态。

## 3. 演示重置基线

执行“重置三名演示学生”后：

- 不存在活动或暂停的旧课时；
- 教师工作台主路径显示“开始 P01 第一课时”；
- 教师从工作台最多两次点击进入 P01 第一课时正式授课；
- 普通使用中的“继续上次授课”保持不变，只调整演示重置后的初始状态；
- 三名学生恢复各自约定的演示学习数据，但没有本轮课堂参与状态；
- 正式测试运行态、课堂活动窗口、投屏游标和设备在线态均恢复到可重新开课的基线。

## 4. 四十五分钟课堂预验收

本轮只覆盖 P01 第一课时的自然教学节奏，不把 P01 第二课时的正式测试和 N04 成果批阅强塞入课堂。

必须依次证明：

1. 教师登录后最多两次点击开始 P01 第一课时。
2. 投屏从产品入口打开并显示同一课堂和同一授课页。
3. 三名学生不输入内部 URL，在两分钟内从学生首页进入课堂。
4. 教师翻页后，投屏和课堂跟随学生在两秒内到达同一页和同一 revision。
5. 学生切换自主学习后，教师翻页不覆盖个人自学游标；学生主动返回跟随时才追到教师当前页。
6. 三名学生完成同一个 canonical 岗位微练习，教师端、学生端和投屏端的参与数、提交数、通过数一致。
7. 错误作答返回针对性反馈和重试入口；重试不丢失历史尝试。
8. 教师暂停后共享推进被禁用，继续后恢复同步；结束课堂后本轮课时只读。
9. 学生返回首页或完整自学时，个人阅读位置与真实练习进度不丢失。
10. 全程没有 5xx、浏览器控制台错误、权限绕过或三端数字冲突。

## 5. 课后真实闭环

课堂预验收完成后，由 `student01` 使用 user-origin 事件完成：

```text
真实前置活动
→ P01 N02 正式测试
→ N04 首次提交
→ 教师退回
→ 学生修改并再次提交
→ 教师确认
→ 学生首页、成果包、能力图谱回流
```

`student02` 和 `student03` 只用于快速预览演示状态，不得替代上述真实状态转换。正式测试与成果闭环可在课堂结束后独立验收，不计入四十五分钟课堂时长。

## 6. 证据与问题分级

不要求连续录屏。每轮保留：

- 关键状态截图；
- 一份最终检查表；
- 失败时的错误页、浏览器控制台输出和相关服务端日志；
- 线上 build-info、SQLite 健康信息和重置结果。

问题分级：

- **P0：** 无法登录、开课、加入、同步、提交、暂停/继续/结束，或发生数据冲突、权限泄漏、跨端状态不一致。立即停止；修复并部署后从重置基线重跑。
- **P1：** 操作含糊、需要试错或明显影响课堂节奏，但存在正常产品路径恢复。记录后可继续完成本轮。
- **P2：** 不阻塞流程的视觉、文案、间距或动画问题。本轮不扩大为视觉重构。

## 7. 线上执行与部署纪律

- 所有预验收、修复后重跑和最终验收均直接针对线上环境。
- 实现仍须先通过相关单元测试、类型检查、结构检查和生产构建，不能用线上试错替代本地工程门禁。
- P0 单独修复、聚焦验证并立即部署。
- P1 每个发布批次最多三项。
- 每次发布执行数据库备份、原子切换、健康检查和失败回滚。
- 每轮结束恢复三名演示学生基线，不把验收过程状态遗留给下一次演示。

## 8. 停止条件

本轮满足以下条件即达到演示级收敛：

- P0 为零；
- 核心课堂路径上的 P1 为零；
- 非核心区域最多保留三个有明确正常规避路径的 P1；
- P2 进入后续清单，不阻塞当前样张交付；
- 线上课堂预验收与 `student01` 课后真实闭环均从重置基线完整通过；
- 线上最终恢复为可直接开始 P01 第一课时的干净演示状态。

## 9. 本轮不做

- 不增加讨论、问答、圆桌、智能导师或补考审批流程。
- 不扩大学生人数。
- 不重做学生首页、教师工作台或能力图谱框架。
- 不用预置演示分数、演示成果或浏览器状态冒充 user-origin 完成事实。
- 不因本轮试教清理 `.git`、SQLite、权威教材、已验证媒体或最终教程截图。
